# results/ — the experiment record

Every run of the evaluation scripts persists its results here. This directory
is **committed to git**: it is thesis evidence, and the ledger is the honest
trial count behind the Deflated Sharpe Ratio (every variant ever evaluated
leaves a row, including the ones that looked bad).

- `ledger.jsonl` — append-only registry: one line per run (timestamp, kind,
  parameters, git commit, headline metrics, artifact path). Never edit or
  rewrite lines; only append.
- `eval_windows/` — one JSON per multi-window evaluation: full per-window
  metrics plus distribution summaries (mean/std/min/median/max), and each
  window's plottable time series.
- `single_run/` — one JSON per single-window backtest: the full metric
  catalog plus a `series` block (per-step timestamps, equity, and aligned
  SP500TR levels; values quantized to cents — metrics are stored at full
  precision separately) consumed by `scripts/plot_results.py`.
- `deflated_sharpe/` — one JSON per deflation (`scripts/deflated_sharpe.py`):
  the strategy's per-period and annualised Sharpe, return skewness/kurtosis,
  the cross-trial Sharpe variance and `n_trials` drawn from this ledger, and
  the resulting Probabilistic and Deflated Sharpe Ratios (Bailey & López de
  Prado 2014). This is where the honest trial count below actually feeds a
  reported number.
- `figures/` — rendered charts (growth-of-1 vs SP500TR, drawdown, per-window
  metric distributions). **Not committed**: every figure is regenerated from
  the JSON artifacts by `scripts/plot_results.py`; commit one explicitly only
  if the thesis cites it.
- `raw/` — verbatim console outputs of runs that predate this directory
  (2026-06-10: the 15-field vs 47-feature stage-1 comparison and the first
  PPO evaluation), preserved for the record.

Score files (`data/scores/*.parquet`) and trained policies (`models/*.zip`)
are **not** committed (licensed-data derivatives / large binaries). Score files
are regenerated by `scripts/train_valuation.py`. The PPO policies belong to an
exploratory RL allocator that was cut from the final system; its training
script is not part of the final codebase, and the JSONs here record the
parameters of those runs for the record.
